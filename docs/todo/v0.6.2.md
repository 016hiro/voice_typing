# v0.6.2 — Push-to-talk 键可配置（原始 scope，已 dropped）

> **状态**：scope locked 2026-04-26，未开工
>
> **主题**：把当前唯一 PTT 键 `Fn` 扩到 5 个 curated 候选（Fn / Right Option / Right Command / F13 / F14）。Settings 加 Hotkey tab + 实时按键检测指示灯 + Reset 按钮。底层把 `FnHotkeyMonitor` 一般化成 `HotkeyMonitor`，事件流签名不变，AppDelegate 状态机零修改。

## 目标

- 用户在 Settings → Hotkey 选 PTT 键，立刻生效
- 5 个候选都能 reliably 工作 in real keyboards：默认 Fn 不变；Right Option / Right Command 在 Apple 键盘外接时通；F13 / F14 在有这些键的外接键盘上通
- Test 指示灯让用户在保存前自验
- 切换 trigger 时若键正被按着，必须 synthesized release 干净退出旧 trigger 状态
- 既有 hands-free / live mode / tap-vs-hold 200ms 阈值与 trigger 选择**完全无关**，自动适用所有 trigger

## 不做

- ❌ Caps Lock：是 toggle 不是 hold；占用会让用户彻底失去全大写输入。等真有需求 v0.6.x 单评
- ❌ 左侧 modifier（左 Cmd/Option/Shift）：在系统快捷键热区，PTT 必冲突
- ❌ 任意组合键（Cmd+Shift+V 等）：长按组合键 UX 别扭
- ❌ 任意单字符键（A、`、Space）：会随便打字误触发
- ❌ Mouse buttons：用户没要求
- ❌ Per-backend / per-mode 独立 hotkey：保持单 PTT 键
- ❌ 任意 keycode 自由输入：避免用户自伤；只暴露 curated 5 个

## 设计决策

### 1. 候选键 = curated 5 个

| 候选 | 检测方式 | 副作用（用户必须知情） |
|---|---|---|
| **Fn**（默认） | `flagsChanged` + `maskSecondaryFn` | macOS emoji 选择被禁 |
| **Right Option** | `flagsChanged` + `maskAlternate` + keycode==61 | 右 Option 字符组合被禁（左 Option 不受影响） |
| **Right Command** | `flagsChanged` + `maskCommand` + keycode==54 | 右 Cmd 系统快捷键被禁 |
| **F13** | `keyDown/keyUp` + keycode==105 | 通常 macOS 无默认绑定 |
| **F14** | `keyDown/keyUp` + keycode==107 | 通常 macOS 无默认绑定 |

不暴露左侧 modifier、任意键、组合键。

### 2. Always swallow 触发键

被选中的键完全归 PTT 用，OS 不再收到该 event。和当前 Fn 行为一致。Settings UI 注明每个选项的副作用文案，让用户自己 trade-off。

### 3. 事件流 API 不变

`HotkeyMonitor.events: AsyncStream<Transition>`，`Transition.{pressed, released}` 沿用。AppDelegate 的 `for await transition in fnMonitor.events` 循环零改动。重命名：`handleFn` → `handleHotkey`，`fnMonitor` → `hotkeyMonitor`。

### 4. Trigger 切换 = synthesized release + swap

```
state.pushToTalkTrigger 变化（用户在 Settings 切了选项）
  ↓
AppDelegate 监听 → 调 hotkeyMonitor.swap(to: newTrigger)
  ↓
swap() 内部:
  ├─ 如果当前键持有 .pressed 状态 → continuation.yield(.released)（让上层取消正在录音）
  ├─ stop()（撤掉旧 CGEventTap）
  ├─ 把内部 trigger 字段换掉
  └─ start()（按新 trigger 装新 mask 的 CGEventTap）
```

避免"切换时正按着键 → recording 永不停"的死结。

### 5. Test 指示灯

Settings → Hotkey 显示一个实时指示灯：用户按下当前选中的 trigger 时圆点亮绿。这是 v0.6.2 的关键 UX guard：F13/F14 不是所有键盘都有，Right Option keycode 区分可能在某些键盘不准；指示灯让用户保存前自验。

实现：HotkeyTab 自己拉一份 `monitor.events` async stream，`.pressed` 时灯亮 200ms。

### 6. UserDefaults 持久化

- Key: `pushToTalkTrigger`
- 类型: String（`HotkeyTrigger.rawValue`）
- 默认: `"fn"`（既有用户首次升 v0.6.2 不感知变化）
- 写入: `AppState.pushToTalkTrigger` 的 didSet
- 读取: `AppState.init` 中读，invalid value 退回 `.fn`

### 7. Reset 按钮

Settings → Hotkey 底部 "Reset to Fn (default)" 按钮。如果用户选了不存在的键（F13 但键盘没这键）保存了 → 录不上音 → 点 Reset 一键回 Fn。escape hatch。

### 8. 不变量 / 兼容性

- 既有用户（已有 UserDefaults）不动 → `pushToTalkTrigger` 缺省为 `.fn` → 行为零变化
- v0.6.1 onboarding flag、HF endpoint cache 等其他 UserDefaults 不受影响
- Hands-free（200ms tap 阈值）、live mode、tap-vs-hold 状态机：trigger-agnostic，自动兼容
- Accessibility 权限：和当前 Fn 路径一样，仍由 `Permissions.checkAccessibility` 弹窗

## 数据流

```
[ User 在 Settings → Hotkey 选 trigger ]
    ↓
[ AppState.pushToTalkTrigger.didSet → UserDefaults 写入 ]
    ↓
[ AppDelegate 监听 state.$pushToTalkTrigger publisher ]
    ↓
[ hotkeyMonitor.swap(to: newTrigger) ]
    │
    ├─ if pressed-held: yield .released
    ├─ stop CGEventTap
    ├─ trigger = newTrigger
    └─ start new CGEventTap (按 trigger 类型装 mask)
         │
         └─ AsyncStream 继续 yield .pressed/.released
              │
              ├─→ AppDelegate.handleHotkey (PTT 状态机不变)
              └─→ HotkeyTab Test 指示灯
```

## 风险

- **Right modifier keycode 区分可靠性**：docs 说 keycode 61 = R-Option / 58 = L-Option，54 = R-Cmd / 55 = L-Cmd，但本仓库没用过这字段。**Spike 任务 #81** 第一步就在 dev 机验证。若不可靠，降级为"any Option / any Command"，文案改写明确，方案不阻塞
- **F13/F14 不是所有键盘都有**：Test 指示灯是用户自验 guard，但用户可能选了不亮就保存。Reset 按钮兜底
- **某些 modifier 在某些 OEM 键盘上 keycode 异常**：罗技 / 雷蛇等可能 remap。Test 指示灯第一道防线，README 注明 "如果你的键盘 keycode 异常，pick Fn 是最可靠的兜底"
- **Live-switch 时 race condition**：用户按住键 + 在 Settings 切 → swap 内部 yield .released 必须在 stop() 前发，否则上层永远收不到。代码里要明确顺序
- **既有用户升级**：UserDefaults 缺省走 `.fn` 路径，不感知变化。验证项放收尾段

## 不变量 / 兼容性

- 当前 Fn 用户升 v0.6.2 行为零变化（默认值 = `.fn`）
- HotkeyMonitor 事件流 API 与 FnHotkeyMonitor 完全一致（`AsyncStream<Transition>`、`.pressed/.released`）
- Hands-free 200ms tap 阈值、live mode、tap-vs-hold 状态机：trigger-agnostic，自动适用
- Accessibility 权限走和当前 Fn 路径完全相同的弹窗
