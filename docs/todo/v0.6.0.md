# v0.6.0 — 发行基础设施（DMG + Sparkle 自更新 + License）

> **状态**：scope locked 2026-04-24，未开工
>
> **主题**：把应用从"本地 dev build" 升到"可分发 + 可自更新"。零功能改动，纯打磨。
>
> 原 v0.6.0 候选「多 backend + streaming 抽象」整体推迟到 v0.7.0+。

## 目标

让一个**完全没碰过这个项目的人**（暂时只有自己）能：

1. 下载一个 `VoiceTyping-0.6.0.dmg`
2. 双击 → 拖进 Applications → 首次右键 "打开" 一次绕过 Gatekeeper
3. 之后所有版本通过 Sparkle 自动检测 + 下载 + 替换 + 重启，无感升级
4. 看到 LICENSE 后明确知道"个人/学习/研究 OK，商用禁止"

## 不做

- ❌ 任何功能改动（多 backend / refine 专版 / 快捷键 / 历史转录 → 全部 v0.7+）
- ❌ Apple Developer ID 证书 / notarization（等稳定再开账号补；Sparkle 路径无需重做）
- ❌ CI 自动化发布（v0.6.x patch 再加；首发用本地 `make release`）
- ❌ beta / stable 分 channel（一个用户 = 单 channel）
- ❌ 公开宣传（依旧自用）
- ❌ 第三方依赖 license 汇编 `THIRD_PARTY_LICENSES.md`（v0.6.x 再补）

## 设计决策

### 1. License: PolyForm Noncommercial 1.0.0

- 为软件量产、plain English、一页纸
- 明确允许 personal / educational / research / nonprofit；明确禁商用
- **不是 OSI open source**（OSI 不允许 field-of-use 限制）—— GitHub 标 "source-available"，符合预期
- 后期想转 MIT / Apache 容易（只你一个 contributor）；反向难，所以**起步 NC 留余地**

### 2. 自更新方案: Sparkle 2

- Mac 自更新事实标准（1Password / Transmission 等无数 indie app 在用）
- SwiftPM 直装 (`from: "2.6.0"`)
- **不依赖 Apple Developer ID**：用 EdDSA 签 update artifact 验证完整性
- Ad-hoc 签名 + EdDSA 路径长期被 Sparkle 1.x 时代的 indie app 验证过；2.x 兼容

### 3. DMG 打包: hdiutil（系统自带）

- 不引入 `create-dmg` 这个 brew 依赖（少一个 dev 环境 setup 步骤）
- DMG 内容简单：`VoiceTyping.app` + Applications 软链接，背景图后期再加
- DMG 本身**不签**（macOS 对未签 DMG 的警告比未签 app 宽松）

### 4. appcast 托管: GitHub Pages (gh-pages 分支)

- URL：`https://016hiro.github.io/voice_typing/appcast.xml`
- 免费 + 稳定 + 不污染 main 分支历史
- DMG 本身托管在 GitHub Releases（流量无限）

### 5. EdDSA 私钥管理

- Sparkle `generate_keys` 生成 keypair
- 私钥存 `~/.voicetyping-sparkle.key`（不进 repo）
- `.gitignore` 加 `*.sparkle_key` 防意外
- **必须备份到密码管理器**：丢了 = 用户没法收任何新 update（要么换 pub key + 让所有用户手装一次）
- 公钥编进 Info.plist `SUPublicEDKey`

### 6. 更新检查策略

- `SUEnableAutomaticChecks=true`：启动时静默 check，命中弹 Sparkle UI
- 菜单栏 "Check for Updates..." 手动入口（`SPUStandardUpdaterController` 自动提供 action）
- Sparkle 默认 24h 一次背景 check（`SUScheduledCheckInterval` 不显式设 = 用默认）

### 7. 首次升级链路必须真实验证

- 发完 v0.6.0 后**立刻**发 v0.6.0.1 dummy（只 bump build number）
- 在装着 v0.6.0 的机器上验证：detect → download → EdDSA verify → replace → relaunch → 版本号变 → TCC（mic + accessibility）授权没丢
- 验证结果写进 `docs/release-process.md` 的 "Known quirks" 段
- **没跑完这一步不算 v0.6.0 ship**

## 数据流

```
 [ 你（dev）]                    [ GitHub ]                [ 用户 App ]
     │                                │                         │
     │  make release                  │                         │
     │  ├─ build .app (ad-hoc)        │                         │
     │  ├─ hdiutil → DMG              │                         │
     │  ├─ sign_update (EdDSA)        │                         │
     │  ├─ update appcast.xml         │                         │
     │  └─ gh release create ─────────┼──→ Release assets       │
     │                                │    (DMG + sig)          │
     │  git push appcast ─────────────┼──→ gh-pages             │
     │                                │    appcast.xml ◄────────┼──  Sparkle 定期拉
     │                                │                         │
     │                                │    DMG ◄────────────────┼──  Sparkle 下载
     │                                │                         │
     │                                │                         │  EdDSA 验签 → 替换
```

无环，3 个 actor 单向数据流。
