- [x] **#B3** ASR 异常高 latency root-cause + fix（done 2026-05-02）
  - 数据（`segment_latency.py` 391 sessions）：v0.6.4 ship 后 ≥5s outlier rate 反而从 v0.6.1 的 2.2% 升到 5.3%。所有 outlier 都是 batch + 1.7B + session-first segment（`startSec=0`）—— 跟 v0.6.4 修之前同形态
  - 根因诊断：v0.6.4 的 keep-alive 代码 wired 正确，但**整段在 dogfood 里大概率没真跑**：(1) `.accessory` LSUIElement app 缺 `ProcessInfo.beginActivity` → App Nap 节流 timer；(2) mac sleep/wake 期间 timer 不发；(3) `Task.detached(priority: .background)` 是最低 QoS，App Nap 状态可能无限 defer；(4) `Log.dev` 在用户没开 Developer logging 时完全 no-op，dogfood 数据没观测
  - 修法：四件套 + 可观测
    - `ASRKeepAlive.start/stop` 加 `ProcessInfo.beginActivity(.userInitiated, reason:)` 抑制 App Nap，stop 时 endActivity 释放
    - 加 `NSWorkspace.didWakeNotification` 监听，wake 时立刻 fire 一次 tick
    - timer callback 内 `Task.detached(priority: .background)` → `.utility`（仍后台 QoS，但有调度保证）
    - 三条 `Log.dev` 升 `Log.asr.notice` —— release dogfood 默认能看到
    - `tickCountSnapshot` 暴露给 `DebugCaptureWriter.Meta.keepAliveTicksAtStart` / `keepAliveTicksAtEnd`，每个 session 落盘
  - 顺手加：`gitCommitSHA` 字段（Makefile 用 PlistBuddy 把 `git rev-parse --short=12 HEAD` 注入 built Info.plist），`Scripts/analysis/keep_alive.py` 脚本，`summary.py` 加 Builds 行
