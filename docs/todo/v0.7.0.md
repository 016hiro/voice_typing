# v0.7.0 — Refine UX overhaul (streaming + Cmd+Z chain)

> **状态**：scope skeleton 2026-05-01，待详细 design doc + decisions。
>
> **主题**：把 refine 从"沉默几秒 → 一次性出字"做成"边生成边出字"的 streaming UX，并顺手收 v0.6.3 punt 过来的 Live × Refine Cmd+Z chain（同样在 injector 路径上，一起做更干净）。
>
> **决策依据**：v0.6.3 dogfood 结论——cloud SSE streaming 已经在 wire 层做了（commit `1621078`），但 refine() 仍累 chunk → 一次性 inject。用户实际看不到流式效果，体验跟 v0.5.x 没差。同期 R9 (Live × Refine Cmd+Z) 也是 injector 路径上的活，一起做能复用 undo group / 增量 paste 设计。

## 目标

- Refine 输出像 ChatGPT 那样在目标 app 里逐字（或逐 chunk）出现，缩短感知延迟
- Live mode 下多段 refine 后按 Cmd+Z 能一次撤回整次录音的 refine（不是只撤最后一段）
- Cloud + Local 两个 refiner 都支持流式（cloud SSE 已就绪；local mlx-swift-lm 走 `streamResponse(to:)`）

## 不做

- ❌ 把 raw-first 路径砍掉（保留作为快速备选；流式跟 raw-first 互斥）
- ❌ 流式期间用户能编辑（被覆盖回写 = 数据丢失），流式期间锁住 capsule 状态 + Esc 取消

## 设计决策（待拍板）

### 1. `LLMRefining` 协议改 API

```swift
// 当前：
func refine(...) async -> String

// 候选 A：AsyncSequence<String>
func refineStream(...) -> AsyncThrowingStream<String, Error>

// 候选 B：callback
func refine(..., onChunk: (String) -> Void) async -> String
```

A 更 Swift 风格。需要 cloud / local 两个实现都改。

### 2. `TextInjector` 增量 paste

三种方案，**待 spike 实测兼容性**：

- **逐 chunk Cmd+V**：每个 chunk 替换 pasteboard 再 paste → 用户原剪贴板被反复污染
- **CGEvent keystroke 模拟**：clean，但慢，非 ASCII 字符（中文）需要 IME-aware 才不卡
- **NSAccessibility insertText**：最干净，但 Terminal / 部分 Electron app 不支持，要 fallback

### 3. Raw-first 流式互斥

raw-first 是"先贴 raw 后台 refine 再 Cmd+Z 替换"。流式 refine 下没有"后台 refine"，必须二选一：raw-first ON → 关流式；流式 ON → 关 raw-first。Settings UI 加互斥逻辑。

### 4. Capsule UI 加流式状态

现在是 `refining` 转圈 → `idle`。流式下应该有 `streaming(progress)` 状态，capsule 显示 "正在写入" + 已输出字数计数。Esc 取消。

### 5. Cmd+Z chain (R9 punt 过来的)

跨 app `NSUndoManager` 不可控。两条路：

- **Injector 加 begin/endUndoCoalescing**：live session 包一层。但 macOS AX/CGEvent 路径下没有 cross-app undo manager
- **段间 paste 紧贴时间窗 system coalesce**：实测 TextEdit / Notes / Cursor 三处 baseline，确认下界
- **退化方案**：live mode 下整段录音结束后再统一 refine + 一次性 replace（折损 progressive 反馈）。最坏退化

调研 1-2 天再实装/退化二选一。

## 风险

- **`TextInjector` 流式 paste 是出 bug 风险最高的代码区**——每个目标 app 都得实测，AX 路径不一致
- **流式期间 focus 切换**：用户流式中切窗口怎么办？停止 + 留半截 vs 强行写完。决策待定
- **mlx-swift-lm `streamResponse` API 稳定性**：upstream 还在迭代，可能在我们这版本周期内 breaking change
- **R9 Cmd+Z 调研可能搞不定**——退化方案"段终统一 refine"会折损 live UX，需要拍板能接受多大折损
