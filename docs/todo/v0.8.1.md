## #E1：Refine-only 文本 CER eval（CI 友好）

**目标**：用公开 ASR-correction 数据集回归 refine 模型 / prompt / backend。完全离线，无需音频，可 CI 跑。

**输入**：`(noisy_asr_text, clean_text)` pair，N=500-2000。

**候选数据集**（待先扫一圈许可证 + 格式）：

- SIGHAN 2013/2014/2015 CSC（中文拼写纠错，公开）
- AISHELL-NER ASR 系列
- ASRU-2019 / Interspeech 系列 paper release 的 ASR-error 数据
- 自造（最坏 fallback）：从 refines.jsonl 抽 noisy text，手标 clean

**指标**：

- 整体 CER（原 vs gold / refine 后 vs gold / Δ = 改善量）
- 分桶 CER（按句长 / 是否含 code-switch / 是否含数字串）
- "改坏率"：refine 引入新 error 的占比

## #E2：E2E pipeline CER eval（自有 captures + 手标 gold）

**目标**：在你自己的真实使用分布上看整条 pipeline 的最终质量。

**为什么不用公开音频数据集**：

- AISHELL / Common Voice / SEAME 的 gold 是 **verbatim**（speaker 说啥写啥），但 clean-up / polish 模式*本来就*要清填充词、补标点 —— verbatim gold 跟 refine 目标错位，refine 越好分数越差
- 域 mismatch：读书音 / 会议音 / 闲聊 ≠ dictation 短句 + 技术词 + code-switch
- Qwen ASR 部分跟官方一样，公开数据集 E2E 测出来 ASR 段是冗余的，只有 refine delta 是新信息——而 refine 评测用 #E1 文本路径更便宜

**Gold 构造**：从 `~/Library/Application Support/.../captures/` 挑 50-100 条 audio，**手敲"我本来想打出来的字"** 作为 gold。这跟 verbatim gold 不一样——含的是用户实际想要的输出（清填充词、补标点、修常见 ASR 错字），跟 clean-up 目标对齐。

**指标**：

- `ASR_CER`：raw ASR 输出 vs gold
- `E2E_CER`：refine 后 vs gold
- `Δ = ASR_CER - E2E_CER`：refine 净贡献（正=帮忙，负=添乱）
- 分桶：按 mode（clean-up / polish）/ 按句长 / 按是否 code-switch
