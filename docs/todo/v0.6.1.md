# v0.6.1 — 中国大陆可用性

> **状态**：scope locked 2026-04-25，开工中
>
> **主题**：让中国大陆用户能装、能更新、能下模型，零配置。两件事：HF 自动镜像兜底 + 首次安装自动 onboarding 默认模型 (Qwen3-ASR 1.7B)。

## 目标

- 中国用户从 DMG 装好 → 启动 → 自动选 mirror（如果 HF 慢/不通）→ 弹 confirm 下默认模型 → 用得上
- 海外用户从 DMG 装好 → 启动 → 自动选官方 → 弹 confirm 下默认模型 → 用得上
- **零 Settings UI、零环境变量、零 token、零账号**

## 不做

- ❌ Settings 里"下载源"开关 —— 自动判定，避免心智负担
- ❌ ModelScope 接入 —— repo ID + 文件结构都要重写映射，等 hf-mirror 真出问题再考虑（v0.6.2+）
- ❌ Sparkle delta updates —— 当前 DMG 40MB，BSDiff 流程的复杂度不值得
- ❌ Bundle 模型进 DMG —— Sparkle 不动 `~/Library/Application Support/`，"代码升级重下模型"问题本身不存在
- ❌ 下载过程实时监控速度 + 中途切 endpoint —— 复杂度高，等 dogfood 出现 "race 选官方但实际下载慢" 的真实 case 再做（v0.6.2+）
- ❌ 首次启动**静默**下载 —— 1.4 GB 是显著流量，必须用户确认

## 设计决策

### 1. HF endpoint：自动 race + bandwidth probe

`swift-transformers` 的 `HubApi` 已经读 `HF_ENDPOINT` 环境变量
（`HubApi.swift:152` `hfEndpointfromEnv()`）。**无需上游 PR**，setenv 即可
全链路生效（WhisperKit + speech-swift 都依赖 HubApi）。

**Race 用 bandwidth probe 而不是 HEAD ping**，因为中国到 HF 经常是
"延迟 OK 但带宽 10 KB/s"，HEAD 测不出来。

并发 GET 一个 ~512KB 已知大小的固定文件（实施时确定具体路径，
建议 `argmaxinc/whisperkit-coreml` repo 里的 stable model file），
5s timeout，算 KB/s。

### 2. 决策表

| 官方 | 镜像 | 选 | 原因 |
|---|---|---|---|
| 失败/超时 | ≥ 100 KB/s | **镜像** | 官方完全不通 |
| 失败/超时 | < 100 KB/s 或失败 | 官方 | 都不通，让真实下载报错 |
| ≥ 100 KB/s | < 5x 官方 | 官方 | 官方够用，差距不显著，优先 source of truth |
| ≥ 100 KB/s | ≥ 5x 官方 | **镜像** | 镜像显著更快 |
| < 100 KB/s | ≥ 100 KB/s | **镜像** | 官方慢得不能用 |
| < 100 KB/s | < 100 KB/s 或失败 | 官方 | 都慢，让真实下载报错 |

**100 KB/s 阈值**：1.4 GB Qwen 1.7B 在 100 KB/s 下要 ~4 小时，下限可接受。
**5x 倍率**：避免边界小幅波动反复切；官方"够用但慢"且镜像快得多才切。
两个阈值都 dogfood 后定稿。

### 3. Cache TTL 不对称

- 选官方：24h（source of truth，稳定）
- 选镜像：6h（误判时尽快回到官方，用户网络变好时快速感知）

### 4. 三级 setenv 策略

| 时机 | 行为 |
|---|---|
| `applicationWillFinishLaunching` (sync) | 读缓存：命中且新鲜 → setenv 缓存值；过期/无 → setenv 默认官方 |
| `applicationDidFinishLaunching` (background Task) | Race → 写新缓存（下次启动生效） |
| 触发首次下载前 (lazy) | `awaitResolutionIfPending(timeout: 3s)` 兜底，首次安装无缓存时用 |

### 5. 隐藏 debug override

```swift
// HFEndpointResolver.applyCachedOrDefault() 第一行：
if let manual = UserDefaults.standard.string(forKey: "voice_typing.hfEndpoint") {
    setenv("HF_ENDPOINT", manual, 1)
    return  // 跳过自动逻辑
}
```

**仅 dogfood / 排查用**，不进 Settings UI、不写进 README。
命令行：`defaults write voice_typing voice_typing.hfEndpoint https://...`

### 6. 首次安装 onboarding：confirm dialog

首次启动检测到默认 backend (Qwen3-ASR 1.7B) 未下载：

```
NSAlert
  title:    "下载语音识别模型"
  message:  "首次使用需要下载 Qwen3-ASR 1.7B (~1.4 GB)。
             下载将在后台进行，可在菜单栏状态查看进度。
             也可稍后在 Settings → Models 手动管理。"
  buttons:  ["立即下载", "稍后"]
```

- 用户点"立即下载" → 触发现有 download 流程（`Manage Models` 里的同一条路径）
- 用户点"稍后" → 不弹了，下次启动也不弹（设 `onboardingDismissedAt`）
- 用户首次下载完成后整个 onboarding flag 自动 retire

**判定"首次启动"**：不存在 `onboardingShownAt` UserDefaults key。

### 7. 不变量 / 兼容性

- 模型仍存 `~/Library/Application Support/VoiceTyping/models/<backend>/`
- Sparkle 升级不重下模型（已经是这样）
- 既有用户（已有缓存模型）：onboarding 检测 `isComplete(.default) == true` → 不弹
- 默认 backend 仍是 `MLXSupport.isAvailable ? .qwenASR17B : .whisperLargeV3`

## 数据流

```
[启动]
  applicationWillFinishLaunching
    ├─ HFEndpointResolver.applyCachedOrDefault()
    │   ├─ 检查隐藏 override → 命中直接 setenv 返回
    │   ├─ 读 cache(URL, age) → 新鲜则 setenv cache 值
    │   └─ 否则 setenv "https://huggingface.co"
    │
  applicationDidFinishLaunching
    ├─ Task.detached { await HFEndpointResolver.resolveAndApply() }
    │       ├─ probe(official) ────┐
    │       ├─ probe(mirror)   ────┼─→ choose() → URL
    │       │                       │
    │       └─ 写 cache + setenv ←─┘
    │
    └─ Task { await maybeShowFirstLaunchOnboarding() }
            ├─ 已 onboarded? → return
            ├─ 默认 backend 已 isComplete? → return
            └─ NSAlert → 用户确认 → 触发 download
                              └─ download 内部：await awaitResolutionIfPending(3s)
                                                 → setenv 终态 → HubApi 走对的 endpoint
```

## 风险

- **hf-mirror.com SLA 不确定**：非官方公益镜像，目前业内广泛使用稳定，
  但无 SLA。Auto fallback 已经包含"两个都失败回官方"，缓解。Dogfood 期观察。
- **测速噪声**：单次 race 可能因瞬时抖动误判，cache TTL 不对称（mirror 6h）做了 dampening；
  hysteresis 复杂化先不上。
- **5x 倍率边界震荡**：例如官方 99 / 镜像 500 (5.05x) → 切镜像；下次官方 105 / 镜像
  480 (4.57x) → 切官方。Cache TTL 已 dampen，可观察。
- **探测文件被 mirror 拒绝同步**：选 stable repo (`argmaxinc/whisperkit-coreml`) 内
  weight 文件，mirror 全镜像同步可能性高。万一失败 race 都失败回退官方。
- **首次 onboarding 用户拒绝下载**：用户体验上等于没下模型，
  Settings → Models 仍可手动 → 跟 v0.6.0 流程一致，无 regression。
