## 副线 #S1：短句 refine skip heuristic

> **状态**：候选优化，v0.7.3 数据驱动。

**动机**：v0.7.3 #B1 telemetry 跑出来 139 次 inline local refine 里 **40% 是 no-op**（input == output），平均每次仍付 ~1700ms LLM round-trip。用户选 light 的本意是想清洁文本，但很多场景 ASR 原文已经够干净——pre-LLM 的 input-side heuristic 应该能把这部分跳过去。

**候选启发式**（任一命中→走 refine；全 miss→直接 inject 原文）：

- 含填充词：`啊` / `嗯` / `呃` / `那个` / `就是` / `这个` / `um` / `uh` / `like` 等出现在词内位置
- 含 stutter：连续重复字符（`我我` / `这个这个` / `the the`）
- 含中文数字串（≥2 char）：`十九` / `二零二四` / `一点二`
- 含中英混合且未加空格：`Filter的` / `commit不commit`
- glossary 命中：dict 里有 entry 的 alias 出现
- 长度阈值：≥ N 字（保险起见，长句即便没明显 marker 也走 refine）

**预期收益**：n=139 数据上，约 56 条 no-op 全部能跳过——节省 ~95s 累计 LLM 时间，p50 降到接近 0（短句直出）。

**风险**：误 skip 会让用户感觉 refine 时灵时不灵。需要先离线 replay 现有 captures 验证启发式 recall（应保留所有 changed 行）vs precision（最大化跳过 noop 行）。
