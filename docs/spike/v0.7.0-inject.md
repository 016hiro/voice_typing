# v0.7.0 #R1 — Stream-inject compatibility matrix

**Goal**: choose the inject path for v0.7.0 streaming refine. Three candidates × N target apps × the failure modes most likely to bite under continuous chunked output.

**Spike harness**: `Sources/VoiceTyping/Spike/StreamInjectSpike.swift`. Triggered from menu bar → "🧪 v0.7.0 Spike". 5 s countdown → injects a fixed 200-char zh+en fixture in `chunkSize=5` chunks at `chunkInterval=50ms`. Remove the harness + menu submenu after this matrix is filled in.

**Fixture**:
```
Hello! 今天测试 v0.7.0 流式 inject。重点观察:
1) Auto-complete 是否吞 chunk
2) 光标位置是否漂移
3) IME 切换是否流畅

示例代码: refineStream(text) -> AsyncStream
微信、Slack、VS Code、Cursor、Terminal 都要测一遍。
```

## How to run

1. Quit any running VoiceTyping. `open build/VoiceTyping.app`.
2. Open a fresh editable region in the target app (empty doc / new chat / new file / new note).
3. Click status bar icon → 🧪 v0.7.0 Spike → pick a method.
4. Within 5 s, click into the target app's text region.
5. Watch the stream. Fill in the row.
6. (Optional) Tail Console for elapsed_ms:
   ```
   log stream --predicate 'subsystem == "com.voicetyping.app" && category == "inject"' --level info
   ```

## Observation legend

- ✅ — fixture arrives intact, nothing weird
- ⚠️ — arrives but with a caveat (record what)
- ❌ — broken (record how)
- — — not applicable / didn't test

## Failure modes to watch

- **swallow** — chunks lost (some characters never appear)
- **autocomplete** — IDE / autocorrect popup eats a chunk or replaces text
- **autospace** — extra spaces inserted (Notion / iOS-style)
- **cursor** — cursor jumps to wrong position between chunks
- **IME** — Chinese input method intercepts and turns chunks into composition candidates
- **paste-clip** — pasteboard ends up holding the last chunk after stream ends (cmdv only)

## Conclusion

**Decision: Cmd+V incremental is the v0.7.0 main inject path. CGEvent and AX are dropped.**

| | Cmd+V | CGEvent | AX |
|---|---|---|---|
| Lossless across all tested apps | ✅ | ❌ Notion data loss | ❌ empty in 5/8 |
| Works in Electron / IDE / Terminal | ✅ | partial | ❌ |
| Works in native NSTextView (TextEdit/Notes) | ✅ | ✅ | ✅ |
| Adds value over Cmd+V anywhere | — | no — slower & lossy | no — same coverage as Cmd+V on its supported set |

Cmd+V is the only candidate that doesn't drop characters in any tested target. CGEvent and AX both have narrower coverage than Cmd+V already provides — neither earns its complexity as a fallback.

### App-specific quirks (carry into #R5 / #R6)

- **Notion**: chunked Cmd+V at 50 ms intervals creates one block per chunk + triggers `>` blockquote markdown. **Mitigation**: deny-list `notion.id` → fall back to batch path (one paste at the end). Hardcoded, not user-facing.
- **Notes**: first line gets auto-promoted to a note title — cosmetic, not a bug. No mitigation.
- **Terminal**: Cmd+V works; user is responsible for not running shell commands while a refine is pasting. No mitigation.
- **TextEdit / VS Code / Cursor**: clean, format preserved, undo coalesces correctly per user dogfood. Default behavior.

### Out of scope (test gaps the decision doesn't depend on)

- **Slack** — user does not use Slack; not retested.
- **微信** — client window rendered black at test time; can't conclude. Cmd+V is the path v0.5+ has used in 微信 for a year without complaints, so the prior is "fine"; verify during v0.7.0 dogfood.
- **Obsidian** — row entries inconsistent (marked ✅✅❌ but note says "not installed"). Treat as untested. Not on the v0.7.0 critical path.

### Performance check

Cmd+V baseline ≈ 2500 ms / 200 chars / 40 chunks → ~12.5 ms wall clock per char, dominated by the 50 ms inter-chunk sleep. For real streaming refine where LLM token cadence is 30–80 ms/chunk anyway, Cmd+V is not the bottleneck.

### Decision feeds

- **#R2** `LLMRefining.refineStream` design — proceeds without conditioning on inject path; protocol returns chunks, injector decides what to do with them.
- **#R5** `TextInjector` incremental API — implement Cmd+V incremental with pasteboard ownership held across the stream (one snapshot at start, one restore at end), IME bypass once at start. Drop the AX / CGEvent branches that R1 hypothetically needed.
- **#R6** AppDelegate refine path — add `bundleID` deny-list check (Notion + future entries) → batch path; otherwise streaming path.
