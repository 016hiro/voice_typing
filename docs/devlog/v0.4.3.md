# v0.4.3 — 回归测试基础设施

**发布日期**：2026-04-20
**状态**：22 unit + 1 E2E comparison test 本地全绿；`make test` / `make test-e2e` 上线；CI 跑 `--skip E2E`；16 条 fixture（10 英文公开语料 + 5 中文公开语料 + 1 用户自录）
**定位**：给 v0.4.2 streaming（和之后所有 ASR 改动）兜底 —— 从零起手搭测试台，不再依赖"每次更新手工复读一遍"。也顺带把 `docs/architecture.md` 对齐到 v0.4.2 最新状态。

## 背景

v0.4.2 ship 后用户点明："每次都依赖我说话输入去测试不太科学" —— 对。之前任何一次改 `transcribe` 路径，验证循环都是 `make build → open → Fn 按住 → 读一段 → 看对不对`。记忆里的基线会漂，边缘 case（长录音、VAD 切段、refiner 失败）几乎不会每次都覆盖。

## 范围（7 项）

1. **Test target + 22 unit tests** — `Tests/VoiceTypingTests/Unit/`，Package.swift 加 testTarget。GlossaryBuilder / CustomDictionary / AppState.tailTruncated 的纯函数覆盖
2. **Fixture 系统** — `Tests/Fixtures/`，`*.wav` + `*.expected.json` 配对；`FixtureLoader.allNames()` 动态扫描，加 fixture 零改测试代码
3. **Fixture 获取脚本三件套**：
   - `Scripts/fetch_english_fixtures.sh`（总协调）拉 JFK + 英文 + 中文
   - `Scripts/fetch_librispeech_fixtures.py`（英文，LibriSpeech dev-clean → 9 条）
   - `Scripts/fetch_fleurs_zh_fixtures.py`（中文，FLEURS cmn_hans_cn dev → 5 条）
   - `Scripts/record_fixture.sh`（自录兜底 —— mic 有电流音时用公开语料更干净）
4. **E2E comparison test** — `E2EASRComparisonTests`，**一次跑 batch + streaming 两路**，打印两边完整转写 + Levenshtein 相似度百分比；代替早期两个独立 test
5. **Makefile `test` / `test-e2e`** — unit-only 快速回归 vs 全量（含 metallib 入 test bundle）
6. **CI 集成** —`.github/workflows/build.yml` 加 `swift test --skip E2E`
7. **架构文档对齐** — `docs/architecture.md` 从 v0.3.0 截止补到 v0.4.2，含 per-app profile（v0.3.1）、API key Keychain + 稳定签名（v0.4.1）、流式自拼 runner（v0.4.2）三次版本的设计决策

## 测了什么、不测什么

### 测

| 层 | 测什么 | 手段 |
|---|---|---|
| `GlossaryBuilder` | Qwen 热词 / LLM 词表 / 关键词匹配 / LRU 预算截断 | 10 unit, 纯函数 |
| `CustomDictionary` | CRUD / 去重 / 软上限 / import-export roundtrip / LRU 更新 | 7 unit, temp 文件 |
| `AppState.tailTruncated` | 短字符串不变 / 长字符串前置 `…` / CJK grapheme 保留 | 5 unit, 纯 static |
| `QwenASRRecognizer` batch + streaming 对比 | 真实音频 → 两路转写 → 打印两边 + Levenshtein 相似度 + 长度边界 + 可选 keyword | 1 E2E, iter 所有 fixture |

### 不测（明确）

- **LLMRefiner** — 非确定（模型采样 + API 抖动）+ 费钱，ROI 低
- **TextInjector / Cmd+V** — 依赖系统焦点 + TCC，测不出
- **Fn hotkey / 权限** — CGEventTap 在 test process 里拿不到 Accessibility
- **Capsule 视觉** — 视觉回归另一个类别
- **Whisper ASR** — 加进 fixture 迭代很容易（换 backend 即可），v0.4.3 先不做，保持 MVP 小
- **Keychain migration** — 会写真实 Keychain，要 mock 得改 production 代码；v0.4.1 已经人工过 migration，后续改动再补

## 关键设计决策

### Batch-vs-streaming comparison，keywords 改成可选

初版 E2E 是两个独立 test class（batch / streaming），每 fixture 对着 `expected.json.keywords` 做 substring containment 断言。第一天跑就踩坑：JFK 的 "ask not" 被 VAD 切成 300ms 独立段，Qwen 幻觉成 "Stop"，streaming 测试 keyword 不包含 "ask not" 失败。

用户反馈：**"测试不应该是转写之后测关键词，而是 batch / streaming 结果都输出，对比 diff"**。重构：

- 单个 `E2EASRComparisonTests.testE2E_Qwen_BatchVsStreaming_AllFixtures`：每 fixture 跑两遍 ASR
- 打印完整两路 transcript 到 stdout（inline）+ 末尾一行总览表（sim% + duration + language + keyword status）
- Levenshtein distance normalised（先去空白+标点+lowercase）→ 相似度百分比
- **相似度纯信息**，没有 hard threshold —— 正常就 100%、VAD 切段标点多了就 95-98%、streaming 短 burst 幻觉就 93%，人眼扫一下能判断
- 硬断言只留：两路都非空、长度在 `[minChars, maxChars]`、streaming partials ≥ `streamingMinPartials`、**keywords 非空时才检查**（给新 fixture 零 keyword 负担）

出的结果一眼看懂：
```
── jfk_en_short (11.0s · en) ─────────────
  batch     [447 ms]            │ And so, my fellow Americans, ask not what your country can do for you...
  streaming [668 ms, 3 partials] │ And so, my fellow Americans. Stop. What your country can do for you...
  similarity                       │ 95%
  keywords                         │ batch ✓  streaming ✓

========================================================================
Batch vs streaming recap (16 fixtures)
========================================================================
  [100%]  Tianshan                       32.2s    zh-CN   ✓
  [ 93%]  fleurs_zh_long_12s_m_1529      12.0s    zh-CN   -
  [100%]  fleurs_zh_med_7s_m_1548        6.8s     zh-CN   -
  ...
```

### `E2E` 前缀 gate

上游 `soniqo/speech-swift` 的 CLAUDE.md 里也是 `E2E` 前缀 + CI `--skip E2E`，对齐。`make test` 过 unit（<20ms）；`make test-e2e` 全量跑。CI 只跑 unit —— 公共 runner 跑不起 1.4GB Qwen + MLX inference。

### Fixture 位置：`Tests/Fixtures/` 不是 test target 内

SwiftPM `.testTarget` 默认把资源视为 `.process` / `.copy`，但那样 fixtures 变成 test bundle 内嵌，改 fixture 要重 build。fixture 是"音频 + 元数据"配对，不该当 swift 产物处理。

解法：不声明为 resource。测试代码通过 `VT_FIXTURE_ROOT` 环境变量拿 root —— Makefile 里硬编码 `$(pwd)/Tests/Fixtures`，CI 用 `${{ github.workspace }}/Tests/Fixtures`。`FixtureLoader.root()` 读环境变量，未设则 `XCTSkip`。

结果：改 fixture 不触发 rebuild；加 fixture 不改 test 代码（`allNames()` 动态扫）；CI / local 路径一致。

### metallib 放进 xctest bundle

MLX shaders 必须和 executable 同目录。Test process 里 `Bundle.main.executableURL` 指向 xctest runner binary，旁边没 metallib。

解法：`make test-e2e` 在 `swift test` 前 `cp .build/release/mlx.metallib .build/arm64-apple-macosx/debug/VoiceTypingPackageTests.xctest/Contents/MacOS/`。`MLXSupport.isAvailable` 加 `overrideAvailable` 测试钩子（`nonisolated(unsafe) static var`），test setUp 读 `VT_MLX_TEST_READY` 环境变量设 true。

没把 metallib 做成 `.process` resource：resource 会被 swift 拷贝到 bundle 的 Resources 子目录，`MLXSupport` 路径检查匹配不上；而且 release-built metallib 和 debug test 的路径关系要手动管理，bash 一行比 SPM 宏更简单。

### `record_fixture.sh` 的 `set -e` 坑

初版脚本用 `set -euo pipefail` —— ffmpeg 被 Ctrl+C 时虽然**把 WAV 正确写完**（你在输出里看到 "Exiting normally, received signal 2" + muxing overhead 摘要），但退出码是 255，bash 立即中断，`expected.json` stub 不会生成。修法：`ffmpeg ... || true`，然后显式检查 `-f "$OUT_WAV" && -s "$OUT_WAV"`。

### 为什么 FLEURS 不是 AISHELL

中文 ASR 公开语料主流是 AISHELL-1（15GB）/ ST-CMDS（8GB）/ THCHS-30（6.4GB）—— 都是单 tarball，小 subset 拿不出来。FLEURS 的 cmn_hans_cn dev 集只 **207MB**，且 HF 上 tarball 直接 URL 可拉，还是 Google 团队精挑的专业播音录音，恰好匹配我们"要少量高质量"的需求。License 都是 CC BY 4.0，commit 到 repo 合法。

## 意外发现（测试立刻抓到）

1. **JFK "ask not" → "Stop"**：Silero VAD 把 300ms 独立 burst 切出独立段，Qwen 幻觉。v0.4.2 devlog 就点过 VAD 切段风险，这次标定了具体例子。Similarity 95%，不算大，但**是唯一一条 >5% 差异的英文短 fixture**，信号清楚

2. **中文长 fixture 相似度 93-95%**：VAD 按静默切段后，streaming 每段末尾加句号，batch 输出是一整句 —— 都是标点层面的差异，去标点后相似度 100%。证明**去标点的 normaliseForDiff 选对了**，否则所有长中文 fixture 都会在 90%

3. **短中文（<10s）相似度 100%**：batch / streaming 完全一致。说明短音频里 VAD 不引入分段差异，streaming 和 batch 收敛
