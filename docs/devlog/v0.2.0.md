# v0.2.0 — 可切换 ASR 后端

**发布日期**：2026-04-16
**状态**：开发中

## 交付的功能

- **三档可切换 ASR 后端**，保持 `SpeechRecognizer` 协议不变：
  - Whisper large-v3（WhisperKit）
  - Qwen3-ASR 0.6B（MLX，4-bit 量化）
  - Qwen3-ASR 1.7B（MLX，8-bit 量化，**新默认**）
- **菜单 Model 子菜单**：每项显示名称 / 预估大小 / 状态（Active / Downloaded / Not downloaded / Downloading N%）。点击即切换，未下载自动触发下载。
- **设置窗口分标签**：
  - **Models** — 三行模型表格，每行 [Switch/Download] 按钮 + 删除按钮 + 真实下载进度条
  - **LLM** — 原 LLM Refinement 配置页
- **真实下载进度**：Qwen 后端接入 `progressHandler` 回调，菜单文字显示 "Preparing Qwen3-ASR 1.7B… 42%"
- **模型文件管理**：
  - 切换后端**不删除**旧模型（下次切回零等待）
  - 用户主动删除：`Manage Models…` 二次确认后清理
  - 删除当前活动后端时，recognizer 重新加载并提示用户
- **迁移路径**：v0.1.0 老用户在 `models/argmaxinc/...` 的 WhisperKit 权重会自动迁移到 `models/whisperkit/argmaxinc/...`，切回 large-v3 无需重下。

## 设计决策

### 默认后端为什么是 Qwen-1.7B

- Qwen3-ASR 1.7B benchmark 上多语言多数据集**优于 Whisper large-v3**，参数更少（1.7B vs 1.55B），量化后磁盘占用 ~1.4 GB vs WhisperKit CoreML 的 ~3 GB
- 0.6B 的 92ms TTFT 在我们"按 Fn → 一次性 utterance"的模式下用不上，留给后续流式版本

### 为什么切换不删除旧模型

- 用户可能想对比不同模型准确率、或临时切回 Whisper 做验证 —— 反复下载 1.4 GB 体验太差
- 磁盘成本由用户决定：显式的 `Manage Models…` 页面让用户自己掌控

### 为什么不抽象出"通用下载器"层

- WhisperKit 的下载进度是黑盒（只能 indeterminate），Qwen 走 `HuggingFaceDownloader` 有细粒度回调 —— 两种模式本质不同，强行统一反而复杂
- 进度传递通过 `SpeechRecognizer.stateStream` 的 `.loading(progress:)` 已经足够，UI 层不关心细节

## 实现期遇到的问题与修复

### Issue 1: `speech-swift` 的 macOS 版本要求

**症状**：README 写的"macOS 14+ / iOS 17+"但 `Package.swift` 里是 `macOS("15.0")`，加依赖后 swift build 直接拒绝 macOS 14 target。

**原因**：`CoreMLASRModel.swift` 用了 `MLState` KV cache（macOS 15+ API），所以整个包 floor 必须 15。README 没同步。

**修复**：把 `Package.swift` `platforms: [.macOS(.v14)]` 升到 `.v15`，同时 `Info.plist` 的 `LSMinimumSystemVersion` 14.0 → 15.0，`swift-tools-version` 5.9 → 6.0（`.v15` 枚举值需要 6.0 PackageDescription）。

**影响**：Intel Mac 和 Sonoma 用户无法升级到 v0.2.0，但本项目 v0.1.0 起就仅 Apple Silicon，且开发机已经是 macOS 26，问题可控。

### Issue 2: Qwen3ASRModel 不是 Sendable，不能直接捕获到 Task.detached

**症状**：Swift 6 工具链下，在 `Task.detached { ... model.transcribe(...) }` 里直接捕获 `let model = self.model` 会报 Sendable 违反。

**原因**：`Qwen3ASRModel` 在上游声明为 `public class`，不是 `final`，不是 `Sendable`。

**修复**：改为捕获 `[weak self]`，在 detached 闭包内重新解包 `self.model`，并用 `transcribeLock: NSLock` 串行化。`QwenASRRecognizer` 类整体标 `@unchecked Sendable`，我们自己承担同步正确性。

文件：[`QwenASRRecognizer.swift`](../../Sources/VoiceTyping/ASR/QwenASRRecognizer.swift)

### Issue 3: MLX 运行需要 `mlx.metallib`，而 SwiftPM 构建不生成

**症状**：`swift build -c release` 成功，app 能启动，但 Qwen 后端在 `prepare()` 阶段抛 `Failed to load the default metallib`。

**原因**：`mlx-swift` 只把 Metal 源码放进 C++ target，**不在 SwiftPM 构建期自动编译成 `.metallib`**。`speech-swift` 仓库提供了一个 helper 脚本 `scripts/build_mlx_metallib.sh`，需要手动调用 `xcrun metal` 编译 33 个 metal kernel 源文件。MLX C++ 运行时按以下顺序查找：
1. `mlx.metallib` 在可执行文件旁边（`Contents/MacOS/mlx.metallib`）
2. `Resources/mlx.metallib`
3. SwiftPM Bundle `default.metallib`
4. `Resources/default.metallib`
5. 编译时 `METAL_PATH` 宏指向的路径

**修复**：Makefile 加 `metallib` target，在 `make build` 时自动调用脚本，把产物放到 `.build/release/mlx.metallib`，然后 `cp` 到 `Contents/MacOS/`。

文件：[`Makefile`](../../Makefile)

### Issue 3.5: Metal Toolchain 不在默认 Xcode 安装里

**症状**：`bash scripts/build_mlx_metallib.sh` 报 `missing Metal Toolchain; use: xcodebuild -downloadComponent MetalToolchain`。

**原因**：Xcode 16/26 之后，Metal shader compiler 作为可选组件，默认不装。

**修复**：Makefile 加 `make setup-metal` target 调用 `xcodebuild -downloadComponent MetalToolchain`。如果这个命令本身因为 Xcode plugin 问题失败（DVTPlugInLoading），需要先 `sudo xcodebuild -runFirstLaunch`（交互式，需要管理员密码）。README 已说明。

**影响**：首次克隆仓库的开发者要做一次性 setup-metal；Qwen 后端加载前会失败并在菜单显示错误，用户可切到 Whisper large-v3 应急。

### Issue 4: 语言代码不兼容

**症状**：调 Qwen 传 `language: "zh"` 识别退化严重。

**原因**：Qwen3-ASR 期望的是**全拼英文单词**（`"chinese"`, `"english"`, `"japanese"`），不是 ISO 639 代码。它把字符串拼进 system prompt token。

**修复**：`Language` enum 新增 `qwenName: String` 属性。`whisperCode` 保留给 WhisperKit 用。

文件：[`Language.swift`](../../Sources/VoiceTyping/Support/Language.swift)

## 验证结果

- ✅ `swift build` + `make build` 都通过
- ✅ `codesign --verify` 通过，app bundle 结构正确（含 swift-crypto/swift-transformers bundles）
- ⚠️ 端到端录音测试**尚未完成**：开发机 Metal Toolchain 缺失导致 Qwen 加载前失败，需用户一次性 `make setup-metal`（可能连带 `sudo xcodebuild -runFirstLaunch`）
- ✅ 设计级验证：Whisper 迁移路径读过代码，默认后端变更 + 启动时迁移不破坏 v0.1.0 用户的 large-v3 文件

## 已知限制

- 切换后端会触发 "Preparing…" 状态，如果按 Fn 录音时后端正在切换，会被拒（"Model still loading…"）
- 第一次下载 Qwen-1.7B 需要 ~1.4 GB，取决于网速需要几分钟
- WhisperKit 的下载进度仍是 indeterminate（上游 API 限制），只有 Qwen 后端有百分比
