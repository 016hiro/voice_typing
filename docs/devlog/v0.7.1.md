- **#B2 Refine prompt 收紧 + 砍 Fix Errors 档**：`RefineMode.conservative` 整段移除（跟 `.light` 重叠），老用户 UserDefaults `"conservative"` 自动迁 `.light`；`light` + `aggressive` 两档 prompt 重写：中文数字回阿拉伯（"九十九" → "99"，量词搭配 "三个文件" 例外）/ 中英混读不要把英文翻成中文 / "这个" 进 filler + 口吃列表 / Polish 的 spoken→written 升 #5 + 显式标为差异点
- **#B3 ASR 异常高 latency root-cause + 修**：v0.6.4 keep-alive 是死代码（dogfood 里被 macOS 节流），重写成四件套——`ProcessInfo.beginActivity(.userInitiated)` 抑制 App Nap、`NSWorkspace.didWakeNotification` 监听补 tick、`Task.detached` QoS `.background` → `.utility`、所有 `Log.dev` 升 `.notice` 让 release dogfood 默认可见
- **观测落盘**：`DebugCaptureWriter.Meta` 加 `keepAliveTicksAtStart` / `keepAliveTicksAtEnd`（每 session snapshot tick 计数）+ `gitCommitSHA`（`make build` 用 PlistBuddy 把 `git rev-parse --short=12 HEAD` 注入 built Info.plist）。dogfood 数据现在能按 commit 切片、并直接验 keep-alive 是否真在跑
- **#B4 partial：分析脚本补齐**——`refine_quality.py` 读 `refines.jsonl` 跑 cloud↔local A/B；`keep_alive.py` 读新 meta 字段按 build 切看 tick coverage，自动 flag 可疑 build；`summary.py` 加 Builds 行 + Refines 行
- **doc drift 收尾**：`README.md:16` 还在描述 v0.7.0 砍掉的 raw-first 模式，换成 streaming refine

## 遇到什么

- **#B3 诊断走过两步**：先用另一个 Claude session 跑 `segment_latency.py` over 391 capture sessions，发现 v0.6.4 ship 后 ≥5s outlier rate 反而从 2.2% 升到 5.3%（v0.6.1 baseline）。fork 到当前 session 接管时，初步假设是"keep-alive 没修对"。code audit 后发现更糟：keep-alive 代码 wired **是对的**，但四个独立的子系统级原因叠在一起让它在 release 里**完全不工作**——App Nap / `.background` QoS 被 defer / wake-from-sleep 不补 tick / `Log.dev` 默认 off 导致前一版数据完全没观测。修这个不是"调整阈值"是"补四块基础设施"
- **`Log.dev` 静默 no-op 是元问题**：v0.6.4 dogfood 数据为空，不是因为 timer 真没跑（虽然事实上确实没跑），而是因为我们**根本没办法知道它跑没跑**。`Log.dev` 的设计原意是"reduce noise"，但用在生产 dogfood 上等于自蒙双眼。这次把 keep-alive 的三条 dev 日志统一升 `.notice`，未来类似的 timer/keepalive 类基础设施都应该默认可观测，再考虑收
- **`gitCommitSHA` 决定不进源 Info.plist**：源 plist 进 git，每次 build 改源 plist 会污染工作树。改用 `make build` 阶段后置注入到 *built* `$(PAYLOAD)/Contents/Info.plist`（PlistBuddy `Add :GitCommitSHA string $$SHA`），源 plist 始终干净

## 为什么这么做

- **#B2 砍 Fix Errors 不补迁移弹窗**：迁移 toast 多 30 行 UI 代码，refine mode 不是关键功能，用户从"严格修错"升到"删 filler / 口吃" 在意外性上低于 silent → off。CHANGELOG 写清楚就够
- **#B3 修法是 prophylactic 同时上 4 件**：每件单独都解释不了 "v0.6.4 outlier rate 反而升高"。比如只修 App Nap 不修 wake hook → 用户合盖再开还是踩坑；只升 QoS 不修 App Nap → timer 还是被节流。dogfood 数据已足以拍板四件一起上，不再单独证伪每条假设以省时间
- **#B5 per-app 热词推到 v0.7.2 单做**：v0.7.1 是 polish 版本，主题是"小 bugfix"。#B5 是 schema migration + UI 重设计的 medium feature，混进来会模糊 release 主题，也让 v0.7.1 周期拖长。单独 v0.7.2 能 `/think` 拍 schema（`ContextProfile.dictionaryFilter` 引用全局子集 vs 独立 store）+ UI 设计
