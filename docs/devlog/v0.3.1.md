# v0.3.1 — Per-app 上下文 profile

**发布日期**：2026-04-19
**状态**：功能完成，待端到端验证

## 交付的功能

### Per-app context profile

按前台 app 的 bundle ID 注入一段 system prompt 片段到 LLM refiner，让模型按 app 调整风格（Slack 偏口语、VSCode 偏代码等）。

**数据模型**：`ContextProfile { id, name, bundleID, systemPromptSnippet, enabled, createdAt }`

**存储**：`~/Library/Application Support/VoiceTyping/profiles.json`，debounced 3 s flush，损坏时 rename `.corrupted-<ts>.json`。Mirror `CustomDictionary` 的架构——已验证过的模式，风险低。

**匹配**：精确匹配 bundle ID（case-insensitive），取第一个 enabled 的 profile。通配符模式留给以后。

**在 pipeline 里的位置**：`AppDelegate.stopRecording` 快照 `frontmostBundleID`，查 profile，把 snippet 传给 `LLMRefiner.refine`；系统 prompt 组装顺序是：
```
baseline (mode.systemPrompt) → profile snippet → glossary
```
普世规则在前，app 风味居中，用户词汇表居末。

**Refiner off 时 profile 不生效**——profile 从不强迫开启 refine。

### UI：Settings → Profiles tab

UI 线程并行重写了整个 Settings 为 Liquid Glass 风格，Profiles tab 镜像 Dictionary tab 的骨架：`SectionCard + Table + 工具条 + sheet editor + alert`。

差异点：
- **"Add…"** 弹 `NSOpenPanel(.application)`，用户从 Finder 挑 `.app`，后台读 `Bundle(url:).bundleIdentifier` + `CFBundleDisplayName`——bundle ID 对用户透明
- **"Add frontmost app"** 一键加当前目标 app（见下面 Issue A 的处理）
- **行渲染**：`NSWorkspace.icon(forFile:)` 实时拉 app 图标；app 被卸载时 fallback 到 generic application 图标
- **Editor**：`TextEditor` 多行 snippet + 底部灰字 monospaced `Matches bundle com.x.y`——bundle ID 可见但不抢戏
- **重复 bundle ID**：第二次 "Add…" 选同一个 app 直接走 Edit 路径不新增，和 store 的 dedup 一致

## 过程中踩的坑

### Worktree 并行开发

v0.3.1 实现和 UI-redesign（胶囊 + Settings Liquid Glass 重写）**同期并行**在两个 thread。一开始我直接在 `ui-redesign` branch 上落 backend 代码，build 挂了——并发写入 `SettingsWindow.swift`。

解决：改用 `git worktree`：
```bash
git worktree add ../voice_typing-profiles -b v0.3.1-profiles main
```
backend 改动全在 `v0.3.1-profiles` 分支的独立 working tree 里。UI 线程 merge 到 main 后，我 `git rebase main`——zero conflict（UI 没动 AppDelegate/LLMRefiner/ContextProfile，只动 SettingsWindow + Capsule* + AppState 的 label 相关段；我动 AppState 另一块 + AppDelegate.stopRecording + LLMRefiner.refine + 新文件）。

结论：**同 repo 双 agent 并行改代码，worktree 是必须**，不是 nice-to-have。

### LaunchServices bundle-ID 碰撞

两个 worktree 各有一份 `build/VoiceTyping.app`，bundle ID 都是 `com.voicetyping.app`。`open <path>` 被 LaunchServices 重定向到"它记得的那份"——不是路径指向的那份。Debug 时看到的 UI tab 数量对不上就是这个原因。

解决：`make install` 到 `/Applications/VoiceTyping.app` 做 canonical；或者 `killall` 后直接 exec `.../Contents/MacOS/VoiceTyping` 绕过 LaunchServices。

### Review 自查发现两处

**Issue A**：`ProfilesTab.addFrontmostApp` 的第一版用了 `NSWorkspace.shared.runningApplications.first { ... }`——order Apple 文档说 "unspecified"，不等于"最近活跃"。注释还写错了说"和 pipeline 一样"（pipeline 用的是 `frontmostApplication` 拿真实前台）。

修法：`AppState` 加 `lastNonSelfFrontmostBundleID`，`AppDelegate` 挂 `NSWorkspace.didActivateApplicationNotification` observer 维护它——每次非-VoiceTyping 的 regular app 激活就更新。启动时从 `frontmostApplication` seed（VoiceTyping 是 LSUIElement，启动瞬间前台不是自己）。"Add frontmost app" 直接读这个值。

**Issue B**：`ContextProfileStore.dedupKey` 用 `.lowercased()`，`lookup` 用 `==`。两端都来自 `Bundle.bundleIdentifier` 实际碰不到差异，但用户手改 JSON 或 import 外来数据会出现 "同一个 bundle ID 存了两份" 的诡异状态。对齐策略：lookup 也走 `.lowercased()`，符合 Apple 的 bundle ID "case-insensitive token" 约定。

## Backend 和 UI 的责任切分（回顾）

```
AppState
  ├── dictionary: CustomDictionary           (v0.3)
  ├── profiles:   ContextProfileStore        (v0.3.1)
  └── lastNonSelfFrontmostBundleID: String?  (v0.3.1, for "Add frontmost app")

AppDelegate.stopRecording
  ├── snapshot frontmostBundleID
  ├── profile = state.profiles.lookup(bundleID: frontmostBundleID)
  └── profileSnippet → both injection paths (waiting-for-refine + raw-first)

LLMRefiner.refine(..., profileSnippet: ...)
  └── compose: mode.systemPrompt + profileSnippet + glossary

SettingsWindow → ProfilesTab
  ├── reads state.profiles.profiles
  ├── mutations via state.upsertProfile / removeProfile
  └── "Add frontmost app" reads state.lastNonSelfFrontmostBundleID
```

## 没做 / 留到以后

- **Bundle ID pattern 匹配**（通配符、正则）：v0.3.1 精确匹配够用。等有用户抱怨再加。
- **Profile 和 Dictionary 的 per-profile 字典覆盖**：现在 profile 只影响 LLM system prompt，不影响字典。想象中有用（某 app 只用特定子集词），但需求不清晰。观察。
- **Profile 模板 / 市场**：分享 profile 给别人。导出已经是 JSON，手动分享够用。
- **Bundle ID picker 的 "Running apps grid"**：NSOpenPanel 够用，SF Symbol + 图标 grid 是 polish。
