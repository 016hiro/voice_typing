# v0.4.4 — VAD 预装 + 界面收敛 + refine 默认 Off + Dev mode

> Small polish release。四件事:
> 1. **Silero VAD 模型打包进 app bundle** — 离线可用,`~1 MB` 首用下载卡顿消失
> 2. **胶囊只显示状态,不再闪实时转写** — 中间态文本从屏上完全移除
> 3. **Refine 档位重命名 + 默认改 Off** — "Conservative" 不再让人误以为不跑 LLM
> 4. **Settings → Advanced tab** — Developer logging 开关 + 一键复制 `log stream` 命令

没有破坏性变更。UserDefaults rawValue 不变,老用户的选择保留。

## 背景

v0.4.2 引入流式 (post-record) 之后暴露两个体验问题:

- **VAD 模型按需下载** — `SileroVADModel.fromPretrained()` 第一次调用会拉 ~1 MB 权重 (HuggingFace `aufklarer/Silero-VAD-v5-MLX`)。用户在离线 / 弱网环境下按下 Fn 会卡死,没有任何进度反馈。对一个"按下就转写"的输入法级工具不可接受。
- **胶囊塞实时文本很丑** — 单行等宽 `tracking(2)` 全大写,中英文混排挤在 520 pt 里,加上每次 partial 都 `tailTruncated(30)` 截头,视觉上抖得厉害。粘贴完又立刻 `hide()`,最终文本闪 <1 s 就没,"根本没什么作用"。

加上一个本来就在的名字坑:

- **`Conservative` 听起来像"保守 = 不跑 LLM"**,但实际是"LLM 跑,但 prompt 严格只修错别字"。真正跳 LLM 的档位叫 `Off`。默认值还是 `Conservative`,导致新用户开箱就吃 LLM 往返延迟而不自知。

## 变更

### 1. Silero VAD 模型打包

#### 实现

权重 (`model.safetensors` 1.2 MB + `config.json` 384 B) 直接塞进 `Resources/SileroVAD/`,Makefile 负责拷进 app bundle:

```makefile
# Makefile:45-54
@if [ -f Resources/SileroVAD/model.safetensors ]; then \
  mkdir -p $(PAYLOAD)/Contents/Resources/SileroVAD; \
  cp Resources/SileroVAD/{model.safetensors,config.json} \
     $(PAYLOAD)/Contents/Resources/SileroVAD/; \
  echo "  embedded Silero VAD → ..."; \
fi
```

运行时检测 + offline 加载 (`Sources/VoiceTyping/ASR/QwenASRRecognizer.swift`):

```swift
fileprivate static func bundledVADCacheDir() -> URL? {
    guard let resourceURL = Bundle.main.resourceURL else { return nil }
    let candidate = resourceURL.appendingPathComponent("SileroVAD", isDirectory: true)
    let weights = candidate.appendingPathComponent("model.safetensors")
    guard FileManager.default.fileExists(atPath: weights.path) else { return nil }
    return candidate
}

// VADActor.get() 内:
if let bundled = QwenASRRecognizer.bundledVADCacheDir() {
    model = try await SileroVADModel.fromPretrained(
        cacheDir: bundled, offlineMode: true  // ← 跳过网络
    )
} else {
    // dev loop (swift run / swift test,没跑 make build)
    model = try await SileroVADModel.fromPretrained()
}
```

`offlineMode: true` 在 upstream `HuggingFaceDownloader.downloadWeights` 里走的分支:

```swift
// speech-swift/AudioCommon/HuggingFaceDownloader.swift
if offlineMode && weightsExist(in: directory) {
    progressHandler?(1.0)
    return  // ← 0 次 HTTP
}
```

#### 为什么不做增量下载 / 延迟加载

- VAD 只有 1.2 MB,跟 app 其他资源 (MLX metallib `~20 MB`、AppIcon `~300 KB`) 一个数量级都不到
- 权重跟代码版本锁定 (silero-vad v5 的 MLX 格式,对应 `SileroVADNetwork` 的 shape),没有"用户想升级 VAD 权重不想升级 app"的需求
- Qwen ASR 模型 (~400 MB-1.4 GB) 才走动态下载路径,因为文件大 + 用户可能切 0.6B/1.7B

#### Attribution

`Resources/SileroVAD/ATTRIBUTION.md` 新增,记录上游 `snakers4/silero-vad` (MIT) + MLX 转换 fork `aufklarer/Silero-VAD-v5-MLX`。

#### Fallback 保留

`bundledVADCacheDir() == nil` 的路径保留了 — dev loop (`swift run` / `swift test` 直接跑,没走 `make build`) 时 `Bundle.main.resourceURL` 指向的不是 app bundle,找不到权重就退回到老行为 (HF 默认缓存 `~/Library/Caches/qwen3-speech/...`)。生产 install 和 CI 两条路都能走。

### 2. 胶囊去文本

这一项走了一次弯路 — 先做了独立的 `TranscriptPreviewWindow` 浮动卡片 (毛玻璃 + 多行 wrap + 1.8s 驻留),实测后觉得 *还是丑* + 注意力分散,直接全砍:

- 删掉 `AppState.capsuleText` (`@Published var`)
- 删掉 `AppDelegate` 里 6 处 `state.capsuleText = ...` mutate
- `labelTextForCapsule` 重命名 `statusTextForCapsule`,简化到只出状态:

```swift
// Sources/VoiceTyping/AppState.swift:155-162
var statusTextForCapsule: String {
    switch status {
    case .idle, .recording: return "Listening"
    case .transcribing:     return "Transcribing"
    case .refining:         return "Refining"
    case .info(let msg):    return msg
    }
}
```

- 同步删掉 `AppState.tailTruncated(_:max:)` 和对应的 5 条 unit test

**流式链路本身没变** — `QwenASRRecognizer.transcribeStreaming` 还在跑,VAD 分段 + 段级 ASR + 增量 yield 都保留。只是 yield 的 partials 不再往 UI 推,`AppDelegate.runASR` 里用 `latest = partial` 记录最新值,最后 return 完整文本给下游 inject。长录音 (>60s Fn 保持) + 段级 cancel 语义都还靠它。

### 3. RefineMode 重命名 + 默认 Off

`Sources/VoiceTyping/LLM/RefineMode.swift`:

```swift
// 默认值翻转
public static let `default`: RefineMode = .off  // was .conservative

// 标签改掉:
case .off:          return "Off"           // 不变
case .conservative: return "Fix Errors"    // was "Conservative"
case .light:        return "Clean Up"      // was "Light"
case .aggressive:   return "Polish"        // was "Aggressive"
```

`shortDescription` 同步改掉,第一句直接点明是否跑 LLM。

**rawValue 不动** (`off / conservative / light / aggressive`),UserDefaults 向前兼容 — 老用户的选择保留,只是菜单上的字变了。新安装首次启动落 `Off`,`llm_ms = -1`。

#### 为什么不一起迁移 raw

理论上可以把 `.conservative` → `.fixErrors` 整个重命名,但需要:
- UserDefaults migration 代码 (old raw → new raw)
- `LLMConfigStore` 写过 `conservative` 字符串的迁移
- 测试覆盖迁移分支

收益只是变量名好看一点。decision: 不做。注释里写清 displayName vs raw 的分离就够了。

### 4. Settings → Advanced tab + Developer logging

新增 `SettingsTab.advanced`,图标 `slider.horizontal.3`。目前只放一样东西:

- **Developer logging** toggle (`state.developerMode`) — 持久化到 UserDefaults `developerMode`,同步到 `Log.devMode` 静态变量
- **Tail the unified log** — 复制 `log stream --predicate 'subsystem == "com.voicetyping.app"' --style compact` 到剪贴板

`Log.devMode = true` 时,`Log.dev(...)` helper 把 verbose 行从默认 no-op 升级成 `logger.notice(...)` 发射,`log stream` 不加 `--level info` 就能看到。已经切过去用 `Log.dev` 的行:

- `Loading Silero VAD from app bundle: /path`
- `Silero VAD not bundled — falling back to HuggingFace cache`
- `Qwen3-ASR loaded: <modelId>`
- `ASR bias: backend=... entries=... context=...` (和 `ASR bias: none`)
- `Context profile: <name> (bundle=...)`

普通用户默认看到: `latency` (notice) + 错误。开 dev 模式后多出上述 5 种 init / per-utterance 诊断行。

**为什么不用 os.Logger 的运行时 level 切换**: 试过写一个 `VTLogger` 结构体包一层,结果 `os.Logger.info(_ message: OSLogMessage)` 的 `OSLogMessage` 必须在调用点是 literal string interpolation,不能作为变量 forward。Swift 编译器会 `error: argument must be a string interpolation`。改走一个 `String` boundary 的 helper (`Log.dev(_ logger: Logger, _ message: @autoclosure () -> String)`),代价是 per-interpolation 的 `privacy: .public` 标记在 helper 里固定成 `.public` — 这个 app 里反正所有打点都是 `.public`,没有 PII,可以接受。

## 耗时打点 (v0.2 就在,本版把级别从 `.info` → `.notice`)

`LatencyTracker` 从 v0.2 就在,每条 pipeline 打一行 `os_log`。v0.4.4 把它从 `Log.app.info` 升级成 `Log.app.notice` — 默认 `log stream` 只给 `notice+`,不升级就要加 `--level info` 才看得到,p50/p95 这种汇总指标不该藏在 debug 里。

```
latency backend=qwen-asr-1.7b mode=off dict=3 rawFirst=false
        asr_ms=812 llm_ms=-1 inject_ms=73 total_ms=920
```

实时看:

```bash
log stream --predicate 'subsystem == "com.voicetyping.app"' \
           --style compact | grep latency
```

近况:

```bash
log show --predicate 'subsystem == "com.voicetyping.app"' \
         --last 10m | grep latency
```

字段:
- `asr_ms` — ASR (batch 或 streaming) 完整耗时
- `llm_ms` — refine 耗时,`-1` = 跳过
- `inject_ms` — 粘贴耗时 (HID tap + 目标 app 响应)
- `total_ms` — 从 `stopRecording()` 到流程结束
- `mode` — refine 档位 rawValue (注意:不是新 displayName,是 enum raw)
- `rawFirst` — 是否 raw-first 策略 (`injectRawFirstThenRefine` vs `injectWaitingForRefine`)

## 非变更 / 下一步

**v0.5 真 live-mic 路线已拍板**: segment-level streaming + batch ASR。upstream `Qwen3ASRModel` 只有 batch `transcribe(audio:) → String`,没有 token-level streaming API,所以 v0.5 走同一套 segment 策略,只是 driver 从"已录完 buffer"换成"mic chunk AsyncStream"。参考 `.build/checkouts/speech-swift/Sources/Qwen3ASR/StreamingASR.swift` 的 `transcribeStream`,基本就是把现有 `QwenASRRecognizer.runStreaming` 的 outer while 循环改成 consume live stream。

### v0.5 需要解决的新问题
- `AudioCapture.maxDuration = 60.0` 硬限要放开 (或改成分段持久化)
- Fn 中途松开时 in-flight 段的 ACK/cancel 语义
- 最后一段没 `.speechEnded` 时的 flush 规则
- Settings UI: streaming 开关能否直接换成"live mic"或分两级 (post-record / live)

## 快速自测步骤

1. `make build && make install`
2. `open /Applications/VoiceTyping.app`
3. Settings → Streaming (experimental) = on,Refinement = Off
4. 在终端开:
   ```bash
   log stream --predicate 'subsystem == "com.voicetyping.app"' \
              --style compact | grep -iE "VAD|latency"
   ```
5. 按 Fn 说一句话 → 松开
6. 应看到:
   - 胶囊只有 `TRANSCRIBING`,没有文字
   - `latency ... mode=off llm_ms=-1 asr_ms=<数字> ...` (notice 级,默认可见)
7. 要看 VAD 初始化那种一次性 info 行 (`Loading Silero VAD from app bundle: ...`),得加 `--level info`:
   ```bash
   log stream --predicate 'subsystem == "com.voicetyping.app"' \
              --style compact --level info | grep -iE "VAD|latency"
   ```
8. 改 Refinement = Fix Errors,再按一次 → `mode=conservative llm_ms=<数字>` (注意 raw 没变)
