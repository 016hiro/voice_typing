## 遇到什么

- **Cloud refine 100% timeout**：dogfood 切 OpenRouter `qwen/qwen3.5-flash-02-23` 时每次 NSURLErrorTimedOut，但 in-app "Test" 按钮通过。两层根因：(a) 8s 默认 timeout 对任何路由层都太紧；(b) 没开 `stream: true`，OpenRouter 心跳没了，reasoning 模型必 buffer 全部 reasoning + visible 才返回。Test 用 "ping → ok" 是 2 token 假阳性。**最终修法是流式 + reasoning.exclude + 60s 默认（reasoning 模型 90s）**，不只是简单 bump timeout
- **DebugCaptureWriter 隐藏 5 个版本的 silent bug**：用户 ls #R8 落盘目录发现连 `audio.wav` / `injections.jsonl` / 完整 meta.json 都没有，只有 `meta.json` (partial) + `segments.jsonl`。回看历史 200+ session 全是这状态。`[weak self]` + writer 只被 pipelineTask 局部 var 持有 → Task 结束时 writer 死前 finalize 闭包还没轮到执行
- **Settings UI cloud 模式溢出 600pt 面板**：之前 #R7 落地时只算了 local 模式（小），cloud 分支（picker + 3 个 LabeledField + Test/Save row）展开后 ~498pt 顶到面板边
- **mlx-swift-lm vs WhisperKit 依赖冲突**：mlx-swift-lm 3.31.3 默认拉 swift-transformers 1.3.0，WhisperKit 0.18.0 钉死 < 1.2.0。手动 pin `"1.1.6"..<"1.2.0"`，验证 1.1.6 已有 `additionalContext` API → 解决
- **Live × Refine Cmd+Z chain (#R9) 没修**：跨 app NSUndoManager 不可控调研未启动，决定 punt 到 v0.7.0 跟流式 refine UX 一起做（streaming UX 是更大的架构变化，scope 拆分）

## 为什么这么做

- **Tier 8/16/24 边界 + 隐藏 8 GB 入口**：8 GB 用户开了本地 refiner 必 swap，UX 灾难。16 GB 给警告 opt-in。24 GB+ 默认仍 OFF（先收 dogfood 数据再考虑改默认），见 [`docs/todo/v0.6.3.md`](../todo/v0.6.3.md) §3
- **R5 偏离 scope（直接方法 vs `.localRefiner` enum case）**：scope 想加进 `ASRBackend`，但 refiner 不是 ASR，会污染那个类型语义。改用 `ModelStore.localRefiner*` 直接方法，5 个新调用点比 enum dispatch 改造小
- **R8 偏离 scope（JSONL vs `refine-<idx>.json`）**：跟 `segments.jsonl` / `injections.jsonl` 一致，复用现有 `_load_jsonl` 分析脚本，少一种格式
- **Refiner 不做 ASR 风格 keep-alive（v0.6.4 走的路）**：refine 本来就有 "等几秒"心智模型，5-30s cold-decompress 可接受；ASR 是 critical path 每次 Fn 都触发。两条路径决策刻意相反
- **Cloud streaming 默认 60s + reasoning 模型 90s**：OpenRouter 边缘 timeout 自己设在 524 触发的几十秒级别；任何 thinking model 没 streaming 都要 30-60s。60s 是流式 + 心跳兜底下的安全垫
- **#R9 punt v0.7.0**：流式输出（用户实际看到字逐渐出现，不是当前的"等 → 一次性贴"）是更大的架构变化（refine API 改 AsyncSequence + injector 增量 paste + raw-first 路径重设计 + capsule UI 改），8-15 小时 + 大量手测。Cmd+Z chain 涉及同样的 injector 路径，一起做更干净
- **Settings UI 修法用扩窗而非压缩内容**：用户拍板"空一点比上下溢出好看"。CLAUDE.md 加硬规则防 Claude 下次再犯（v0.5.1..v0.6.3 至少 3 次溢出 regression）
- **DebugCaptureWriter 写 regression test**：`do { writer.finalize(...) }` 块强制 writer 局部 ref 退出作用域，poll 文件系统验证 finalize 工作完成。Pre-fix 代码会 fail。这是从"silent failure"改成"明确测出来"的关键
