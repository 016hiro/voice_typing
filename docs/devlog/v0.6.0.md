# v0.6.0 — 可分发 + 可自更新 + 显式 LICENSE

> Ship date: 2026-04-24

## 背景

v0.5.x 一直是 "clone repo + `make build` 才能用" 的状态。v0.6.0 把分发链路补齐：双击 DMG 就能装，之后所有版本通过 Sparkle 自动检测 + 下载 + 替换 + 重启。零功能改动——把 v0.5.3 没正式 release 过的 Hands-free + Writer 修一并打包进首个分发版本。

目标：

1. **能装**：DMG 拖进 Applications 即可（首次右键打开绕一次 Gatekeeper）
2. **能升**：之后所有版本无感，不需要重新下载 + 重新拖
3. **License 明确**：source-available，禁商用
4. **不依赖 Apple Developer 账号**：等稳定后再开账号补公证

## 决策

### 1. License: PolyForm Noncommercial 1.0.0（/think 回合）

候选评估：

| 候选 | 适用性 |
|---|---|
| MIT / Apache | 不限商用——和需求反向 |
| CC BY-NC | Creative Commons 官方建议**不要用于软件**（不处理专利、源码/二进制差异）|
| BSL（Business Source License）| "延迟转 OSS" 模型，对个人项目过重 |
| **PolyForm Noncommercial 1.0.0** | 为软件量产、plain English、一页纸、Heather Meeker 起草 |

PolyForm NC 不是 OSI open source（OSI 不允许 field-of-use 限制），GitHub 标 "source-available"。这正是想要的语义。后期想转 MIT 容易（一个 contributor），反向难——**起步 NC 留余地**。

### 2. 自更新方案: Sparkle 2（/think 回合）

候选评估：

| 候选 | 评估 |
|---|---|
| **Sparkle 2** | Mac 自更新事实标准（1Password / Transmission 等），SwiftPM 直装，**EdDSA 签 update artifact 不依赖 Apple Developer 账号** |
| 自己拉 GitHub Release API 比版本号 | 实现简单，但每次"看到新版"= 用户重新手装 DMG → 抵消本版本目标 |
| 全 GitHub Actions 自动化发布 | 加 1-2 天调试，CI 里管 EdDSA 私钥额外风险——**v0.6.x 再做** |

Sparkle 2 走 EdDSA 签名验证 update 完整性，这条路径长期被 Sparkle 1.x 时代的 indie app 用过。我们 ad-hoc 签名（已有的 `make setup-cert` 自签证书）+ EdDSA 签 update artifact = 完整链路，不需要 Apple notarization。

### 3. 跳过 notarization 的代价（明确接受）

| 环节 | 有 Developer 账号 | 没有（ad-hoc + EdDSA） |
|---|---|---|
| 首次安装 | 双击 → 拖 → 直接打开 | 双击 → 拖 → **右键 → 打开**（首次绕过 Gatekeeper 一次）|
| 后续启动 | 丝滑 | 一次绕过后记住，不拦 |
| 自更新替换 | Sparkle 替换即生效 | Sparkle EdDSA 验签后本地 FS 替换，**不重新打 quarantine** |
| TCC 授权稳定性 | 稳定 | 稳定（`make setup-cert` 已保证 cdhash 稳）|

接受首次手动右键一次的代价。等稳定 + 真要对外分发时再开账号 + 补公证——Sparkle 配置基本不用动。

### 4. DMG 工具: hdiutil（/think 回合）

不引入 `create-dmg` 的 Homebrew 依赖（少一个 dev 环境 setup）。`hdiutil create -srcfolder ... -format UDZO` 系统自带，DMG 内容简单：`VoiceTyping.app` + `/Applications` 软链接。背景图等以后想做再加。

### 5. appcast 托管: GitHub Pages（gh-pages 分支）

URL：`https://016hiro.github.io/voice_typing/appcast.xml`

- 免费 + 稳定 + 不污染 main 分支历史
- DMG 本身托管在 GitHub Releases（流量无限）
- 用 `git worktree add --orphan` 一条命令建好长期 worktree 在 `build/gh-pages/`，`make release` 直接写到这

## 实现细节

### Sparkle 集成（`66`）

```swift
// AppDelegate.swift
let updaterController = SPUStandardUpdaterController(
    startingUpdater: true,
    updaterDelegate: nil,
    userDriverDelegate: nil
)

// StatusItemController.swift — 菜单项
let item = NSMenuItem(
    title: "Check for Updates…",
    action: #selector(SPUStandardUpdaterController.checkForUpdates(_:)),
    keyEquivalent: ""
)
item.target = updaterController
```

配置全在 Info.plist（`SUFeedURL` / `SUPublicEDKey` / `SUEnableAutomaticChecks`）。零代码 UI——Sparkle 自带的 dialog 够用，不重做。

### Sparkle.framework 必须 ditto 进 bundle（**实测踩到**）

SwiftPM `swift build -c release` 会产 `.build/release/Sparkle.framework/`，但**不会自动 embed 到 .app 的 `Contents/Frameworks/`**。binary 链接 `@rpath/Sparkle.framework/Versions/B/Sparkle`，运行时找不到 → dyld crash。

修法：`make build` 前置 `ditto $(SPARKLE_FRAMEWORK) $(PAYLOAD)/Contents/Frameworks/Sparkle.framework`。`ditto` 比 `cp -R` 更靠谱，保留 `Versions/B` 符号链接 + 完整保留 XPC Services / Updater.app 的代码签名结构。`codesign --force --deep` 后 `codesign --verify --deep` 通过，Updater.app + Installer.xpc + Downloader.xpc 都 validate。

写进 `gotchas.md` 防忘。

### EdDSA keypair 管理

- Sparkle `generate_keys` 生成的私钥默认存 macOS Keychain（account name `ed25519`），不落盘
- `sign_update` 默认从 Keychain 读私钥，零额外配置
- `generate_keys -x` 可导出 private key 文件 → **必须备份到密码管理器**（丢了 = 所有用户没法收 update，除非换 pubkey + 让所有人手装）
- `.gitignore` 加 `*.sparkle_key` / `sparkle_private_key.txt` 防意外 commit

### `make release` 流程（`69`）

```
make release VERSION=0.6.0 BUILD=14
  ├── make build              ← swift build -c release + ditto Sparkle.framework + codesign
  ├── make dmg                ← hdiutil → build/VoiceTyping-0.6.0.dmg
  ├── make setup-sparkle-tools (idempotent)
  ├── sign_update -p <DMG>    ← EdDSA 签，输出 base64
  ├── update_appcast.py       ← 注新 <item> 进 build/gh-pages/appcast.xml
  └── 打印 next-step 指令     ← gh release create + cd build/gh-pages && git push
```

`make release` 本身只产本地 artifact——不 push、不创 release。最后两步（公开操作）由人工执行，避免误操作。
